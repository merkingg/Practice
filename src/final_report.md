# Финальный отчёт второго вариативного задания по проектной практике
## Telegram-бот «ПолиКот советует»

---

## Хронология работы над проектом

### Этап 1 — Выбор темы и постановка задачи

На начальном этапе была определена тема проекта: создание Telegram-бота для поиска книг. Сформулированы основные требования:

- Бот должен искать книги по жанру, автору и рейтингу
- Интерфейс — на русском языке
- Данные должны быть актуальными и обновляться в реальном времени, а не быть заранее прописаны в коде

Был выбран стек технологий: **Python + python-telegram-bot + Google Books API**.

---

### Этап 2 — Исследование технологий

Проведено исследование доступных API для получения данных о книгах. Рассмотрены Google Books API, Open Library, ЛитРес и Goodreads. По итогам анализа выбран **Google Books API** как наиболее доступный и актуальный вариант для учебного проекта.

Изучена документация библиотеки **python-telegram-bot v21.5**, в частности:

- `InlineKeyboardMarkup` и `CallbackQueryHandler` — для кнопок
- `ConversationHandler` — для многошагового диалога
- `Application.run_polling()` — для запуска бота

---

### Этап 3 — Первоначальная разработка

Написана базовая версия бота со следующими возможностями:

- Главное меню с тремя кнопками
- Поиск по жанру (10 жанров)
- Поиск по автору (фиксированный список 10 авторов)
- Поиск по рейтингу (5 категорий)
- Отображение 5 книг с описанием и ссылкой на Google Books

**Проблема на этом этапе:** бот возвращал сообщение «книги не найдены» из-за фильтра `langRestrict=ru` в запросах к API.

---

### Этап 4 — Исправление поиска

Выявлена и устранена причина отсутствия результатов: параметр `langRestrict=ru` ограничивал выборку до книг с русскоязычными метаданными, которых в Google Books крайне мало.

Параметр удалён, поисковые запросы переработаны на английские фразы (`fiction novels`, `mystery thriller` и т.д.).

---

### Этап 5 — Добавление автоперевода

Поскольку описания книг в Google Books API преимущественно на английском, подключена библиотека **deep-translator** для автоматического перевода названий и описаний на русский язык.

```bash
py -m pip install deep-translator
```

Добавлено предупреждение пользователю о времени ожидания (~30 секунд).

---

### Этап 6 — Получение API-ключа Google Books

В процессе тестирования обнаружена ошибка **429 (Quota Exceeded)** — дневной лимит запросов был исчерпан. Для увеличения лимита получен API-ключ через Google Cloud Console (Books API, раздел Credentials).

Ключ добавлен в команду запуска как переменная окружения `GOOGLE_BOOKS_API_KEY`.

---

### Этап 7 — Модификация: свободный ввод автора

Фиксированный список из 10 авторов заменён на свободный ввод имени пользователем. Реализован с помощью `ConversationHandler`:

1. Пользователь нажимает «По автору»
2. Бот просит ввести имя
3. Пользователь пишет любое имя
4. Бот ищет книги через API

Добавлена дополнительная фильтрация: проверка, что найденные книги действительно написаны введённым автором (а не просто содержат это имя в названии).

---

### Этап 8 — Улучшение навигации

Обнаружена проблема: при нажатии кнопки «Главное меню» исчезала последняя книга из подборки (кнопка была прикреплена к сообщению с книгой и редактировала его).

Решение: кнопка «Главное меню» теперь отправляется отдельным сообщением после всех книг.

---

### Этап 9 — Модификация: фильтрация нежелательных книг

В результатах поиска появлялись энциклопедии, кроссворды, справочники и учебники. Добавлен список стоп-слов `EXCLUDED_WORDS` — книги с такими словами в названии или категории исключаются из выдачи.

---

### Этап 10 — Модификация: кураторские списки книг

Категории «Новинки» и «Бестселлеры» давали нерелевантные результаты через обычный поиск API. Созданы два кураторских списка по 25 книг:

- `NEW_BOOKS` — актуальные новинки 2024-2026 годов
- `BESTSELLER_BOOKS` — мировые бестселлеры всех времён

Каждый раз бот случайно выбирает 3 книги из списка, что создаёт эффект разнообразия при повторных запросах.

Книги хранятся на английском языке (для корректного поиска через API), но отображаются пользователю в переведённом виде.

---

### Этап 11 — Обработка ошибок API

В ходе тестирования зафиксированы две основные ошибки:

| Ошибка | Причина | Решение |
|---|---|---|
| 429 Too Many Requests | Исчерпан дневной лимит запросов | Ждать до следующего дня или использовать API-ключ |
| 503 Service Unavailable | Временная недоступность API | Повторный запрос с упрощённым поисковым запросом |

Добавлена обработка ошибки 503 с автоматическим повторным запросом.

---

### Этап 12 — Настройка бота в BotFather

Через **@BotFather** настроены:

- Аватар бота
- Описание: «Привет! Я — ПолиКот, твой персональный книжный штурман»
- Команды бота: `/start`

---

### Этап 13 — Развёртывание на Railway

Для обеспечения постоянной работы бота (не только при запущенном терминале) настроено развёртывание на облачном сервере **Railway**:

1. Создан репозиторий на GitHub
2. Загружены файлы: `bot.py`, `requirements.txt`, `Procfile`
3. Подключён репозиторий к Railway
4. Добавлены переменные окружения: `TELEGRAM_BOT_TOKEN`, `GOOGLE_BOOKS_API_KEY`

Бот начал работать бесперебойно.

---

### Этап 14 — Тестирование с пользователями

Бот протестирован с реальными пользователями. Выявлена проблема: бот не отвечал на `/start` после удаления функции `start`. Добавлен обработчик команды `/start` в функцию `main()`.

---

## Итоги проекта

### Что реализовано

- Поиск книг по 10 жанрам через Google Books API
- Поиск по любому автору (свободный ввод)
- Поиск по 4 категориям рейтинга
- Автоперевод названий и описаний на русский язык
- Кураторские списки новинок (25 книг) и бестселлеров (25 книг)
- Фильтрация нежелательных книг (энциклопедии, справочники)
- Постоянная работа на сервере Railway

### Технические решения

| Задача | Решение |
|---|---|
| Перевод описаний | deep-translator (Google Translate) |
| Свободный ввод автора | ConversationHandler |
| Актуальные новинки и бестселлеры | Кураторские списки + случайная выборка |
| Фильтрация мусора | Список стоп-слов EXCLUDED_WORDS |
| Постоянная работа | Развёртывание на Railway |

### Сложности и способы их преодоления

| Сложность | Решение |
|---|---|
| `langRestrict=ru` не работает | Убран параметр, запросы переведены на английский |
| Описания на английском | Подключён deep-translator |
| Лимит API 429 | Получен API-ключ в Google Cloud Console |
| Нерелевантные результаты | Фильтрация + кураторские списки |
| Исчезновение книги при нажатии кнопки | Кнопка вынесена в отдельное сообщение |

### Возможные улучшения в будущем

- Поиск по ключевому слову (свободный ввод)
- Пагинация — кнопка «Показать ещё»
- Сохранение избранных книг в базе данных
- Рейтинговая система — пользователи могут оценивать книги
